Swarm Tab Redesign

Three-level drill-down: Swarm History → Session Topology → Agent Detail

A

Mirror Sessions Tab (Recommended)

Left sidebar lists all past swarm runs (ID, topology, agent count, timestamp). Clicking one shows the topology canvas + agent list in the right panel. Clicking an agent opens a detail drawer with type, role, metrics, and full communication log — same pattern as the Sessions tab.

Pros

  • Consistent UX with Sessions tab
  • Users already know the pattern
  • Left sidebar scales to many swarm runs

Cons

  • Canvas topology gets less horizontal space
B

Dropdown + Full-Width Canvas

A dropdown at the top selects the swarm run. Full-width canvas below shows topology. Below that, a scrollable agent list with expandable rows for communication logs.

Pros

  • Maximum canvas space for topology visualization
  • Clean when there are few runs

Cons

  • Doesn't scale well with many swarm runs
  • Different pattern from Sessions tab
C

Three-Column Layout

Left: swarm run list. Center: topology canvas. Right: agent detail panel (appears on click). All three visible simultaneously.

Pros

  • No context switching — everything visible
  • Fast agent comparison

Cons

  • Cramped on smaller screens
  • Each column fights for space
