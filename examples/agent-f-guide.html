Monograph

Build the
knowledge graph.

Before agents touch a single line of code, Monograph maps every file, import, and dependency into a queryable graph. Agents know exactly what changes what before they act.

monograph: index the codebase
$ npx monomind@latest monograph build --codeOnly

createorg

One org per
project. Every
project.

createorg provisions a dedicated autonomous organization for a repository. It links Linear for task intake, GitHub for execution, and AWS for test and deploy infrastructure.

createorg: provision project org
$ npx monomind@latest createorg \
--name agent-f-api \
--repo github.com/agent-f/api \
--linear-team API \
--aws-profile agent-f-prod \
--spec-driven
✓ Org "agent-f-api" created with 5 agents
✓ Linear webhook registered (team: API)
✓ GitHub App installed on agent-f/api

The Agents

Five agents.
One shared
goal.

Each org gets a team of five specialized agents. They share memory via AgentDB, coordinate through the swarm, and each has a distinct read/write surface.

◆
PM
product manager
Watches Linear for new issues. Writes specs. Breaks epics into tasks. Assigns work to the team. Nothing ships without a spec.
reads / writes
Linear specs
◉
Architect
system designer
Reads the spec. Queries Monograph for blast radius. Designs the solution: files to create, APIs to change, DB migrations needed.
reads / writes
specs graph
⌨
Developer
implementer
Implements the architect's design. Writes code, runs linters, opens the PR. Uses TDD London School: tests first, implementation second.
reads / writes
design GitHub
⊛
Tester
qa / infra
Runs the full test suite against AWS ephemeral environments. Reports failures back to Developer. Validates deploy before merge.
reads / writes
AWS test results
◎
Reviewer
code quality
Reviews the PR. Checks security, performance, correctness. Either approves or sends back with specific inline comments.
reads / writes
PR diff review
