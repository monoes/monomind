Getting Started

From zero to autonomous in five minutes

Install Monomind, wire it into Claude Code, and run your first autonomous agent loop.

1. Installation
# Option A: global install npm install -g monomind # Option B: npx (always latest) npx monomind@latest --version
2. MCP Setup

The MCP server requires the explicit mcp start subcommand. Auto-detect is disabled by default (set MONOMIND_MCP_AUTODETECT=1 to restore legacy behavior).

claude mcp add monomind npx monomind@latest mcp start # Verify tools are registered claude mcp list
3. Project Initialization
cd your-project npx monomind init # Interactive wizard: topology, memory backend, hooks npx monomind daemon start # Starts background workers (security, health, learning...) npx monomind doctor --fix # Checks: Node 20+, git, config, daemon, DB, MCP, disk
4. First Session
claude — live session
5. Autonomous Loop
# Research → Build → Review until nothing left /mastermind:autodev --tillend # 9 improvements, then stop /mastermind:autodev 9 --tillend # Review loop until clean /mastermind:review --tillend --auto

Concepts

Memory & Knowledge

Four-tier memory palace, HNSW vector indexing, BM25 search, and Monograph — the code knowledge graph with 23 MCP tools.

Memory Palace — Four Tiers
L0
In-flight
Current session context. Stored in drawers.jsonl. Each drawer has a key, value, tags array, timestamp, and score. Ephemeral — not persisted across sessions unless promoted.
L1
Working
Cross-session working memory. Closets store aggregated entries from multiple drawers. Closet boost adds +0.5 per matching search term. BM25 parameters: K1=1.5, B=0.75.
L2
Long-term
Persisted embeddings. AgentDB with HNSW indexing. Hybrid backend combines SQLite for exact lookups with AgentDB for semantic search. Pure-JS HNSW gives O(log n) approximate nearest neighbor vs O(n) brute-force.
L3
Shared
Cross-agent namespace. PartitionedHNSW shards by agent domain. Federation allows swarm agents to read each other's L3 without owning writes. Knowledge graph edges land here.
Storage Backends
BackendUse CaseNotes
SQLiteBackendSmall projects, single-agentDefault. No external deps.
AgentDBBackendVector semantic searchPure-TypeScript HNSW index.
HybridBackendProduction (recommended)SQLite + AgentDB per ADR-009. Auto-routes by query type.
PartitionedHNSWMulti-agent L3Domain-sharded; agent-scoped writes.
DiskAnnBackendVery large datasetsDisk-resident ANN index.

Learning Pipeline
Step 1
RETRIEVE
Pure-JS HNSW ANN search across L2/L3 tiers. Returns top-K neighbors by cosine similarity.
Step 2
LOG
Trajectories, steps, and command/route outcomes are recorded (intelligence.ts, command-outcomes.ts, route-outcomes.ts).
Step 3
MEASURE
Recommended routes are correlated with actual outcomes; doctor reports routing accuracy and adherence.
Step 4
CONSOLIDATE
Dedup, detect contradictions, and prune old patterns from patterns.json. The neural judge/distill loop lives on the monoes-full-loop branch.

Swarm Coordination

Six topologies, five consensus algorithms, and a 15-agent domain hierarchy — from a single coordinator to a Byzantine fault-tolerant hive.

Topologies
Hierarchical
Queen → domain leads → workers. Best for complex feature development. Default.
Mesh
Peer-to-peer, all agents equal. Best for distributed analysis with no single authority.
Hierarchical-Mesh
Queen manages peer clusters. Recommended for large swarms.
Adaptive
Reconfigures topology dynamically. High overhead — use for long-running sessions.
Centralized
Single coordinator, fan-out workers. Fastest for homogeneous parallel tasks.
Star
Hub-and-spoke. Testing swarms — one orchestrator, N independent runners.
Consensus Algorithms
AlgorithmFault ToleranceBest For
Raftf < n/2Default. Strong consistency, single leader. Production code changes.
Byzantinef < n/3Untrusted agent environments, security-critical decisions.
GossipEventualLarge swarms where propagation latency is acceptable.
CRDTPartition-tolerantConcurrent state merges without coordination (offline agents).
QuorumMajorityRead-heavy workloads with strict majority agreement.

Concepts

Intelligence

Deterministic keyword routing, route-outcome measurement, and trajectory + pattern logging — running automatically in the background. No neural training in the lean build.

How it works
Keyword Routing
createKeywordRouter maps a task description to an agent type by keyword scoring — no LLM call and no learned weights. Deterministic and sub-millisecond.
Route-Outcome Measurement
Recommended routes are auto-correlated with actual outcomes. doctor surfaces routing accuracy and recommended-vs-actual adherence over a rolling window.
Trajectory + Outcome Logging
Steps and trajectories (intelligence.ts) plus command and route outcomes (command-outcomes.ts, route-outcomes.ts) are recorded as plain JSON for later analysis.
Pattern Consolidation
The learning worker dedups, detects contradictions, and prunes old entries from patterns.json on a 30-minute interval.
CLI
# Inspect logged patterns npx monomind neural patterns --action list # Predict routing for a task from logged patterns npx monomind neural predict --input "implement auth" # Optimize the pattern store npx monomind neural optimize --method quantize npx monomind neural optimize --method compact # Routing accuracy / adherence npx monomind doctor

The full neural learning loop — SONA self-optimizing architecture, LoRA adapters, EWC++ memory preservation, Flash Attention, MoE routing, and the RL agent suite — lives on the monoes-full-loop branch and is not available in the lean build.

Mastermind

18 autonomous operation commands with Brain Load/Write protocol, --tillend loop mechanics, and the autodev research-build-review cycle.

autodev — The Autonomous Build Loop
Phase 1
Research
Parallel: git log, file scan, package.json, README, TODO/FIXME grep, monograph god nodes, memory search for already-built items. Returns ranked list of 3–5 improvement candidates.
Phase 2
Selection
Picks best candidate by feasibility × blast-radius × focus alignment. Stores selection to AgentDB memory. Focus flags: --focus security, --focus dx, --focus performance.
Phase 3
Build
Invokes mastermind:build with concrete spec, acceptance criteria, and blast radius guard. Spawns architect → coder → tester → reviewer agent chain.
Phase 4
Review Loop
Invokes mastermind:review inline. Runs Code Reviewer + Security Engineer + Reality Checker in parallel. Repeats up to 5 iterations until zero findings.
Phase 5
Log
Records completion to memory. If count > 1, continues to next improvement. If --tillend, schedules next session via ScheduleWakeup.
--tillend Loop Mechanics
Session 1: /mastermind:autodev --tillend → runs → schedules ScheduleWakeup(delay=60s) Session 2: wakeup fires → checks staleness guard → if --rep N < state.currentRep: SKIP (stale wakeup) → else: run → schedules next wakeup Session N: empty round (zero findings AND zero actions) → emits loop:complete → removes .monomind/loops/{id}.json → STOP # Stop a loop manually touch .monomind/loops/{loop-id}.stop
Universal Flags
FlagDefaultPurpose
--tillendoffRepeat until empty round (zero findings, zero actions)
--repeat <N>offRepeat exactly N times
--wait <seconds>60Minimum seconds between repeat runs
--maxruns <N>50Safety cap for --tillend
--autooffNo confirmation prompts
--confirmoffAsk before each action
--focus <area>—Bias toward: security, dx, performance

MonoFence AI

AI manipulation defense for LLM applications. Detects prompt injection, jailbreaks, role switching, context manipulation, encoding attacks, and PII exposure using 50+ patterns — sub-millisecond latency.

Install
# Standalone package — works with any LLM stack npm install monofence-ai
Quick Usage
import { createMonoDefence } from 'monofence-ai'; const fence = createMonoDefence(); // Fast check — sub-millisecond, >12,000 req/s const result = await fence.detect(userInput); if (!result.safe) { console.log(result.threats); // threat types detected console.log(result.overallRisk); // 0.0 – 1.0 } // Scan LLM output for PII leakage or echo attacks const scan = await fence.scanOutput(llmResponse, originalPrompt); // Multi-turn context tracking console.log(fence.getContextState()); // 'clean' | 'probing' | 'escalating' | 'attack' fence.resetContext(); // start a new conversation
Key Features
🔍
EvasionDetector
Normalizes obfuscated inputs before pattern matching. Defeats homoglyph substitution (Cyrillic/Greek lookalikes), spaced characters (i g n o r e), leet substitution (ign0re), and base64 encoding. Applied automatically in every detect() call.
📈
ContextTracker
Sliding-window escalation state machine for multi-turn conversations. States progress clean → probing → escalating → attack. In attack state, overallRisk is raised to at least 0.5 even on individually benign inputs. Enabled by default.
🔎
OutputScanner
Scans LLM output for PII leakage, echo attacks (output that mirrors injected instructions), policy violations, and contradictions with the original prompt. Call fence.scanOutput(llmOutput, prompt).
✅
Allowlist
5 built-in bypass rules plus user-defined runtime rules with TTL decay. Full-bypass rules short-circuit detection entirely; typed rules allow detection to run for other threat types. Narrows false positives for trusted inputs.
🔒
SecurityHook
Registers a pre-task and pre-command scanner with priority 1000. Aborts execution when escalation state reaches attack. Off by default — opt in via SecurityHook.register() or the CLI flag.
🤝
SecurityConsensus
Aggregates threat assessments from multiple parallel agents with attention-based weighting. A single critical threat short-circuits scoring (fail-secure). Returns consensus, confidence, and criticalThreats.
CLI Integration
# Wire MonoFence AI into the pre-task hook (blocks on 'attack' state) npx monomind hooks pre-task --monofence-ai-check # Enable in mastermind:review — adds security pass to each review iteration /mastermind:review --monofence-ai-check # Deep security scan (includes SecurityConsensus across parallel agents) /mastermind:review --monofence-ai-check --monofence-ai-security-deep
