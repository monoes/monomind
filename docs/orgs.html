.monomind/orgs/<name>.json

Define your organization

A plain JSON file: goal, roles, hierarchy via reports_to, and a policy block per role. No wizard step — author it directly, or start from the shipped sample.

.monomind/orgs/content-team.json
{ "name": "content-team", "goal": "Build and publish 3 blog posts per week", "roles": [   { "id": "boss", "type": "boss", "reports_to": null },   { "id": "writer", "reports_to": "boss",     "policy": { "denyTools": ["bash"] } },   { "id": "seo", "reports_to": "boss" } ] }   # then: $ monomind org run content-team  
Hierarchy via reports_to
No separate topology field — the boss role has reports_to: null, everyone else points at who they report to. Flat or deep, your call.
Per-role policy, enforced
Each role's policy block gates tool access, file read/write scope, web access, and token budget — checked on every tool call, with a full audit trail.
Real SDK sessions
Each role is a real Claude Agent SDK session — not a simulated Task-tool subagent inside your current conversation. It runs independently, in the background.
Cross-process discovery
--cross-process (default on) lets orgs running in different monomind processes — even different projects — discover and message each other.
Live dashboard — :4243

Communication patterns

Any shape, via reports_to

There's no separate topology field — hierarchy is just each role's reports_to value. Here's what a few common shapes look like.

pattern: mesh
Mesh
All agents communicate directly. Best for small, tight collaborations where everyone coordinates with everyone else.
Best for 1–3 roles. Quick iteration, no bottleneck coordinator.
pattern: star
Star
Boss at center. All roles report directly. Optimal for 4–6 agents, with central coordination and no middle management overhead.
Best for 4–6 roles. Direct reporting, fast feedback loops.
pattern: hierarchical
Hierarchical
Boss at top, middle manager(s) in between, executors at leaf level. Scales to large orgs with many concurrent roles and clear chains of command.
Best for 7+ roles. Scales coordination without overloading the boss.
